# 기능 테스트 커맨드

Pull Request 생성 전에 구현된 기능을 종합적으로 테스트하는 명령어입니다.

## 프로세스

1. **기능 타입 식별**
   - 프론트엔드, 백엔드, 또는 풀스택 여부 판단
   - 파일 변경사항을 확인하여 범위 파악

2. **프론트엔드 테스트** (해당되는 경우)
   - Playwright MCP를 사용한 E2E 테스트
   - 테스트 시나리오:
     - 사용자 상호작용 플로우
     - 폼 검증
     - UI 컴포넌트 렌더링
     - 접근성 준수
     - 반응형 디자인
   - 필요시 스크린샷 캡처

3. **백엔드 테스트** (해당되는 경우)
   - REST API 엔드포인트 테스트
   - 방법:
     - 적절한 도구를 사용한 직접 API 호출
     - 응답 코드 확인 (200, 400, 401 등)
     - 응답 데이터 구조 검증
     - 에러 케이스 테스트
   - 관련된 경우 데이터베이스 작업 확인

4. **테스트 케이스 커버리지**
   - ✅ 정상 작동 (happy path)
   - ✅ 에러 케이스 (잘못된 입력, 네트워크 오류)
   - ✅ 경계 조건
   - ✅ 사용자 시나리오 end-to-end

5. **테스트 리포트 생성**
   - 모든 테스트 결과 문서화
   - 실패 또는 이슈 기록
   - 권장사항 제공

## 출력 형식

```
🧪 기능 테스트 중: EMBED-XXX

📱 프론트엔드 테스트 (Playwright):
✅ 사용자 로그인 플로우 - 통과
✅ 폼 검증 - 통과
✅ 에러 메시지 표시 - 통과
✅ 접근성 체크 - 통과
⏱️  테스트 소요 시간: 12.3초

🔧 백엔드 테스트 (API):
✅ POST /api/auth/login - 200 OK
✅ 잘못된 자격증명 - 401 Unauthorized
✅ 필드 누락 - 400 Bad Request
✅ 토큰 생성 - 확인됨

📊 전체 결과:
✅ 모든 테스트 통과 (8/8)
⚠️  경고 1개: 로딩 상태 추가 고려
❌ 실패 0개

💡 다음 단계:
- 모든 테스트 통과! PR 생성 준비 완료
- /finish-ticket 실행하여 워크플로우 완료
```

## 테스트 전략

### 프론트엔드 (Playwright)
- 기능으로 이동
- UI 요소와 상호작용
- 예상 동작 확인
- 콘솔 에러 체크

### 백엔드 (API 테스트)
- 각 엔드포인트 테스트
- 요청/응답 확인
- 에러 핸들링 체크
- 데이터 지속성 검증

## 결정 지점

- **모든 테스트 통과**: `/finish-ticket` 실행
- **테스트 실패**: 이슈 수정 후 `/test-feature` 재실행
- **부분 통과**: 경고 문서화, 차단 여부 결정

## 다음 단계

테스트 성공 후:
- `/finish-ticket` 실행하여 PR 생성 및 병합
